Creating Reports with Microsoft Word

By Neil Avent

Producing the user interface

Now we have decided to use Word to handle the reporting for us we need to take a look at what it will need to do before we get coding. I appreciate that this is a lot of waffle before we get to the nitty-gritty of programming, but a clear idea of what the program will do beforehand can prevent so much hassle, especially on larger projects.

We can either have your program create a new Word document from scratch each time a report is generated, or modify a template document to give the correct information. For this scenario we will use a template so we can have a look at other Word functions than just typing text into it.

I have included a word document for you to download. It is called Report.doc and is very basic but gives us something to work on at the moment. If you load it up and take a look you will spot references throughout it: e.g. USERID01 and Q23. These will be used by our project as pointers to amend the corresponding text for each question we ask. Before the values are inputted into the template you can do whatever you want to the document, feel free to make the text bold, italic or colored. Styling can be applied to the pointers as well.

When you have finished spicing up the template we can start creating the code to input the values. Create a new Standard EXE project and add these controls to the form. I know there is a lot to input so you are probably better off downloading the example than creating it yourself. For those who want to create them though here is a list:

Hopefully you should end up with something like this:

Now let's start adding code!

Private Sub Form_Load()
    HScroll1_Change
End Sub

Private Sub HScroll1_Change()
    lblScroll = HScroll1.Value
End Sub

Private Sub HScroll1_Scroll()
    HScroll1_Change
End Sub
